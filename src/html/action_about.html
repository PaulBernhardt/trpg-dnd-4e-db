Original Prototype

It all started with online Character Builder.
I was, and still is, spending 3 hours on buses each day. Sometimes I spent them on the original, offline Character Builder.
Until it went online. Wanting better offline access, the prototype as seen on right was born.
It was a bookmarket that just grabbed entries from online compendium into a popup, applied DataTables on top of it, and you can then manually save the popup for offline use.
It searched so much faster then online, even at home, that I can forget CB and build characters using a goal-oriented approach, instead of browsing through options.
Screenshot date: 2010-11-17.

1st Generation

So I kept working on it.
Inspired by DataTables' search code, I created a search term parser that works with google like syntax. Now we can search for feat with "bonus to damage" OR "extra damage" -"feat bonus". Something you cannot do online.
New features are also being added. Preview panel, multi columns, or pick out items and generate a simple sheet.
Look matters, too, so the script will grab a nice wallpaper through Coral, output fancy CSS styles, data uri icons etc.
Needless to say, it helped me a lot in the two years that follows, both as a DM and as a player.
Screenshot date: 2012-03-23. Main development ceased in 2011 March.

2nd Generation

The problem with first generation is that it's designed for computers.
Consider that "Powers" is a 15MB html file, and the browser has to layout all its contents, then create a 8800 rows table. My relatively new (by then) XT720 Android phone won't even load it.
Thus come the second generation of the offline compendium, powered by the prototype jQuery Mobile. It looks really good. Impressive animation, elegance style, familiar icons. An index was created from Ails' RPG icons, data are processed in multiple threads, and appcache used to preload data.
Turns out it is a disaster. First, Android browser refuse to load local HTML. Second, cookie and web workers are not supported offline. Google really want you stay online.
There are workarounds, but the biggest problem is, jQuery Mobile takes very long time to display entry titles (rightmost screen). A few times longer then displaying their _contents_ in previous version.
This is when I admit defeat and give up on it. At least the index were reused for first version. Screenshot date: 2012-02-17.

3rd Generation Prototype

Third generation development started in 2012-09-09, and is a bold experiment in what can JavaScript do. Instead of letting others do most of the work, the new version do everything by script.
Sadly, when a prototype is working, Firefox pulled the plug on enablePrivilage and killed it with no hope of resurrection.
After playtesting D&D Next for a while, work restarted in May when a replacement notebook arrives.
With a simplified database structure, the second prototype works like a charm. From here begins the long journey of turning it into a polished product.
Screenshot date: 2013-05-16.
